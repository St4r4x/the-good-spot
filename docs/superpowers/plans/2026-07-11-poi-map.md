# POI Map Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Ajouter des points d'intérêt (POI) filtrables par groupe sur la carte de The Good Spot, affichés dans toute la zone commune calculée, avec clustering pour éviter la surcharge visuelle.

**Architecture:** Un nouvel endpoint backend `GET /pois?bbox=...&groups=...` fait un seul appel Geoapify Places (`filter=rect:...`) et regroupe les résultats par groupe de catégories. Le frontend calcule le bbox de la zone commune (Turf.js), filtre précisément les POI reçus avec `isPointInPolygon` (déjà existant), et les affiche sur la carte via `leaflet.markercluster` (nouvelle dépendance) avec une icône Lucide par groupe.

**Tech Stack:** FastAPI + httpx (backend), Next.js 16 / React 19 / Leaflet / Turf.js / lucide-react (frontend), pytest + respx (tests backend), Vitest (tests frontend).

## Global Constraints

- Spec de référence : `docs/superpowers/specs/2026-07-11-poi-map-design.md`.
- 8 groupes de catégories, mapping Geoapify exact (vérifié en live, ne pas
  modifier sans revérifier) :
  - `education`: `education.school`, `childcare.kindergarten`, `education.music_school`
  - `sport`: `sport.fitness`, `sport.pitch`, `sport.sports_centre`, `sport.horse_riding`, `activity.sport_club`
  - `commerce`: `commercial.supermarket`, `commercial.convenience`, `commercial.food_and_drink`, `commercial.marketplace`
  - `health`: `healthcare.hospital`, `healthcare.clinic_or_praxis`, `healthcare.pharmacy`
  - `parks`: `leisure.park`, `leisure.playground`
  - `catering`: `catering.restaurant`, `catering.cafe`, `catering.bar`
  - `public_transport`: `public_transport`
  - `culture`: `entertainment.culture`, `entertainment.museum`, `entertainment.cinema`, `tourism.sights`
- Libellés par défaut si `name` absent (backend renvoie `name: null`, le
  frontend applique le libellé) : Éducation → « École », Sport → « Lieu de
  sport », Commerce → « Commerce », Santé → « Établissement de santé »,
  Parcs → « Parc », Restauration → « Restauration », Transports en commun →
  « Arrêt de transport », Culture → « Lieu culturel ».
- `filter=rect:lon1,lat1,lon2,lat2`, `limit=500` (max autorisé par
  l'API Geoapify, vérifié en live).
- Aucun appel `/pois` si aucun groupe n'est coché.
- Recalcul complet à chaque changement de groupes cochés ou de zone —
  pas de cache incrémental par groupe.
- Icônes Lucide vérifiées disponibles dans `lucide-react` installé :
  `GraduationCap` (Éducation), `Dumbbell` (Sport), `ShoppingBasket`
  (Commerce), `Stethoscope` (Santé), `Trees` (Parcs), `UtensilsCrossed`
  (Restauration), `BusFront` (Transports en commun — distinct de `Bus` déjà
  utilisé pour le mode de transport), `Landmark` (Culture).
- Marqueurs POI neutres (fond `var(--card)`, bordure `var(--border)`,
  icône `var(--muted-foreground)`) — pas de nouvelle couleur fonctionnelle,
  conforme à la One Accent Rule de `DESIGN.md`. Les clusters utilisent
  `var(--primary)` (teal) à la place des couleurs vert/jaune/orange par
  défaut de `leaflet.markercluster`, qui violeraient cette règle.
- Transitions 150–200 ms + `motion-reduce:` ; `cursor-pointer` sur tout
  cliquable ; cibles tactiles ≥44 px ; jamais couleur seule pour un état.
- Commandes de vérification (depuis `frontend/`) : `npm run test`,
  `npm run lint`, `npm run build`. Depuis `backend/` (avec le `.venv` du
  projet activé) : `python -m pytest tests/ -q`.

```python
def validate_groups(groups: list[str]) -> None:
    unknown = set(groups) - POI_GROUPS.keys()
    if unknown:
        raise HTTPException(400, f"groups inconnu(s) : {sorted(unknown)}")


def parse_bbox(bbox: str) -> str:
    parts = bbox.split(",")
    if len(parts) != 4:
        raise HTTPException(400, "bbox doit contenir 4 valeurs : lon1,lat1,lon2,lat2")
    try:
        [float(p) for p in parts]
    except ValueError as exc:
        raise HTTPException(400, "bbox doit contenir des nombres") from exc
    return bbox


def group_for_categories(categories: list[str], groups: list[str]) -> str | None:
    # Iterate POI_GROUPS' canonical (insertion) order, not the client-supplied
    # `groups` order, so a category overlapping two groups always resolves the
    # same way regardless of how the frontend ordered its query param.
    for group in POI_GROUPS:
        if group in groups and any(cat in categories for cat in POI_GROUPS[group]):
            return group
    return None
```

À la fin du fichier, ajouter :

```python
@app.get("/pois")
async def pois(bbox: str, groups: str) -> dict:
    validated_bbox = parse_bbox(bbox)
    group_list = groups.split(",")
    validate_groups(group_list)
    categories = ",".join(cat for g in group_list for cat in POI_GROUPS[g])

    async with httpx.AsyncClient(timeout=30) as client:
        resp = await client.get(
            PLACES_URL,
            params={
                "categories": categories,
                "filter": f"rect:{validated_bbox}",
                "limit": 500,
                "apiKey": GEOAPIFY_API_KEY,
            },
        )
        resp.raise_for_status()
        results = []
        for feature in resp.json()["features"]:
            props = feature["properties"]
            group = group_for_categories(props.get("categories", []), group_list)
            if group is None:
                continue
            lon, lat = feature["geometry"]["coordinates"]
            results.append(
                {"lat": lat, "lon": lon, "name": props.get("name"), "group": group}
            )
        return {"pois": results}
```

```ts
  bbox: [number, number, number, number],
  groups: PoiGroup[]

    bbox: bbox.join(","),
    groups: groups.join(","),
  });
```

```ts
  commerce: "Commerces du quotidien",
  health: "Santé",
  parks: "Parcs & nature",
  catering: "Restauration",

  health: Stethoscope,
  parks: Trees,
  catering: UtensilsCrossed,
  public_transport: BusFront,
  culture: Landmark,
};

const DEFAULT_NAMES: Record<PoiGroup, string> = {
  education: "École",
  sport: "Lieu de sport",
  commerce: "Commerce",
  health: "Établissement de santé",
  parks: "Parc",
  catering: "Restauration",
  public_transport: "Arrêt de transport",
  culture: "Lieu culturel",
};

export function poiBbox(zone: PolygonFeature): [number, number, number, number] {
  const [minLon, minLat, maxLon, maxLat] = turf.bbox(zone);
  return [minLon, minLat, maxLon, maxLat];
}

export function poisInZone(pois: Poi[], zone: PolygonFeature): Poi[] {
  return pois.filter((p) => isPointInPolygon([p.lon, p.lat], zone));
}

export function poiLabel(poi: Poi): string {
  return poi.name ?? DEFAULT_NAMES[poi.group];
}
```

```tsx
"use client";

import type { PoiGroup } from "@/lib/api";
import { POI_GROUPS_ORDER, POI_GROUP_ICONS, POI_GROUP_LABELS } from "@/lib/pois";
import { cn } from "@/lib/utils";

type PoiFiltersProps = {
  selected: PoiGroup[];
  onChange: (groups: PoiGroup[]) => void;
  disabled: boolean;
  error: string | null;
};

export function PoiFilters({ selected, onChange, disabled, error }: PoiFiltersProps) {
  function toggle(group: PoiGroup) {
    onChange(
      selected.includes(group) ? selected.filter((g) => g !== group) : [...selected, group]
    );
  }

  return (
    <div className="flex flex-col gap-3 border-t border-border px-4 py-4">
      <h2 className="text-sm font-semibold text-foreground">2 · Points d&apos;intérêt</h2>
      {disabled && (
        <p className="text-xs text-muted-foreground">
          Calculez d&apos;abord la zone commune avec vos deux lieux de travail.
        </p>
      )}
      <div className="flex flex-wrap gap-2">
        {POI_GROUPS_ORDER.map((group) => {
          const Icon = POI_GROUP_ICONS[group];
          const isSelected = selected.includes(group);
          return (
            <button
              key={group}
              type="button"
              aria-pressed={isSelected}
              disabled={disabled}
              onClick={() => toggle(group)}
              className={cn(
                "flex min-h-11 cursor-pointer items-center gap-1.5 rounded-lg border px-3 text-sm transition-colors duration-150 focus-visible:ring-3 focus-visible:ring-ring/50 motion-reduce:transition-none disabled:cursor-not-allowed disabled:opacity-50",
                isSelected
                  ? "border-primary bg-primary/10 text-foreground"
                  : "border-border bg-background text-muted-foreground hover:bg-muted"
              )}
            >
              <Icon aria-hidden className="size-4" />
              {POI_GROUP_LABELS[group]}
            </button>
          );
        })}
      </div>
      {error && (
        <p role="alert" className="text-sm text-destructive">
          {error}
        </p>
      )}
    </div>
  );
}
```
